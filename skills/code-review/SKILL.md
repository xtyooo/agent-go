# Code Review Skill

当用户要求你审查代码、检查实现质量、排查潜在问题时使用本技能。

## 工作方式

1. 先明确本次审查的代码范围和用户关注点。
2. 优先输出问题，而不是先总结优点。
3. 每个问题都应包含文件位置、风险说明和建议修复方向。
4. 重点关注：
   - 运行时错误
   - 并发或取消语义
   - 数据一致性
   - 安全问题
   - 缺失测试
5. 如果没有发现明确问题，应说明没有发现阻塞问题，并列出剩余测试风险。

## 输出格式

使用中文输出，结构如下：

- Findings：按严重程度列出问题。
- Open Questions：列出需要用户确认的问题。
- Tests：说明已验证和未验证的内容。
